# Trace Viewer

The Soroban Debugger VS Code extension includes a built-in **Trace Viewer** that lets you import and browse execution traces previously exported by the CLI.

## Exporting a Trace (CLI)

When running a debugging or execution session in the CLI, you can export the full execution footprint (arguments, storage state, budget, and events) to a JSON file using the `--trace-output` flag.

```bash
soroban-debug run \
  --contract token.wasm \
  --function transfer \
  --args '["Alice", "Bob", 100]' \
  --trace-output my_trace.json
```

## Importing a Trace in VS Code

Instead of reading the raw JSON, you can browse it interactively in the editor:

1. Open the Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`).
2. Type and select **Soroban: Import Execution Trace**.
3. Select the JSON file generated by the CLI.

The extension will process the file and populate the **Execution Trace** view inside the Soroban Debugger sidebar. This renders a read-only tree detailing the contract summary, budget usage, storage diffs, events, and the full call sequence.
